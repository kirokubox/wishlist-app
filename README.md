# 買いものリスト

欲しい気持ちを一度置いて、納得して買うための自分専用Webアプリです。

「買わせないための節約アプリ」ではなく、欲しい気持ちを否定せず、後から冷静に判断して納得して買うためのリストです。初回の登録だけ軽くちゃんと行い、継続的な入力はほぼゼロになるよう設計しています。

## 画面構成（4タブ）

- **ほしいもの**：欲しいものを「テーマ名だけ」で即登録。発売日と候補の有無から「発売予定／候補あり／テーマだけ」へ自動分類する。やめたものは件数があるときだけ末尾へ表示する
- **補充**：日用品を品名だけで登録。「買った」を押すと一度消えて、既定3ヶ月後（品目ごとに変更可）に自動で再表示される
- **買った**：購入日・価格・満足度5段階・一言を見返す。満足度はあとから追記できる
- **設定**：JSONエクスポート／追加型JSONインポート（v1・v2・v3形式を読み込み可）／Markdownエクスポート

「ほしいもの」の状態を手作業で振り分ける欄はありません。編集画面では任意の発売日を設定でき、発売日を過ぎた品も、買うかやめるまでは「発売済み」として残ります。

## データの保存場所と注意

このアプリはサーバーを使いません。ログイン機能も外部データベースもありません。

- テキストデータはブラウザ内の localStorage（キー：`yuki-wishlist-data`）に保存されます
- v1・v2からv3へ初回変換するときは、変換前のlocalStorage文字列を日付付きの別キーへ一度だけ退避します
- 候補の写真はブラウザ内の IndexedDB（`yuki-wishlist-photos`）に保存されます
- **写真はJSONバックアップに含まれません。** 端末内だけの参考用です。消えて困る写真は端末の写真アプリにも残してください
- PCとスマホでデータは自動同期されません。移行はJSONエクスポート／インポートで行います
- ブラウザのサイトデータを削除すると、データも写真も消えます。週1回程度のJSONバックアップを推奨します
- JSONやMarkdownには個人的な内容が含まれるため、GitHubや公開フォルダに置かないでください（`.gitignore` でバックアップ既定名は除外済み）

## 開発環境での起動方法

```powershell
npm.cmd install
npm.cmd run dev
```

`http://127.0.0.1:5173/wishlist-app/` をブラウザで開きます。

本番ビルド確認：

```powershell
npm.cmd run build
```

## 公開（GitHub Pages）

`main` ブランチへpushすると、GitHub Actions（`.github/workflows/deploy.yml`）が自動でビルドしてGitHub Pagesへデプロイします。

- 公開されるのはアプリ本体（HTML / CSS / JavaScript / manifest / アイコン / Service Worker）だけです。登録したデータや写真はGitHubには保存されません
- `robots.txt` と `noindex` で検索避けをしていますが、絶対的なアクセス制限ではありません。公開URLをSNSやnoteに載せないでください

## PWA（ホーム画面追加）

Android Chromeで公開URLを開き、メニューから「ホーム画面に追加」すると、アプリのように起動できます。

Service Workerは、デプロイごとにキャッシュ名へビルド版番号を刻み、ネットワーク優先＋新バージョン検知時の自動リロードで「更新したのに古い画面が残る」問題を防いでいます（かぞえ帳と同じ構成）。

## 技術構成

- React + TypeScript + Vite（単一 `App.tsx` ＋ `styles.css`）
- localStorage（テキスト）＋ IndexedDB（写真Blob、長辺1280pxへJPEG圧縮）
- PWA（manifest / Service Worker）
- GitHub Pages + GitHub Actions

## 意図的に入れていないもの

- ECサイト連携・価格自動取得
- 通知・リマインダー
- クラウド同期・ログイン
- 必要度・優先度などの細かい入力項目（初期版にあったが、入力を軽くするため削除）
- 発売通知・予約済み・到着待ちなどの追加状態
- 手動の区分変更・並べ替え
